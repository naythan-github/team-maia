# Quality Assurance Checklist

**REMINDER: Follow core non-negotiable standards in CLAUDE.md**
- Australian English spelling (optimise not optimize, centre not center, analyse not analyze)
- Sentence case formatting for all headings and bullet points
- Never merge experiences - each bullet must be from a single exp_id
- Use exact database metrics only - never create or calculate new figures

---

## Pre-Submission Quality Checks

### **Core Standards Verification**
- [ ] **Australian English spelling** used throughout (optimise, centre, analyse, realise)
- [ ] **Sentence case formatting** applied to all headings and bullet points
- [ ] **No American English variants** (organize, center, analyze, realize)
- [ ] **Proper nouns capitalised** correctly (Azure, AWS, Microsoft, etc.)

### **Database Integrity Checks**
- [ ] **Each bullet traces to single exp_id** - No merging or combining
- [ ] **Exact metrics used** - No invented or calculated figures
- [ ] **No duplicated exp_id** between Key Achievements and Professional Experience
- [ ] **Bullet allocation matches** database rules (Zetta: 5, Telstra: 7, OneAdvanced: 7, Viadex: 2, Halsion: 2)

### **Content Quality Verification**
- [ ] **Power Verb Framework applied** - Strong action verbs showing leadership
- [ ] **Quantified results included** - Specific metrics demonstrating business impact
- [ ] **Strategic focus maintained** - Business value beyond task completion
- [ ] **Cross-functional collaboration** emphasised where relevant
- [ ] **Stakeholder level indicated** - Scope showing seniority of engagement

### **ATS Optimisation Checks**
- [ ] **JD keywords naturally integrated** - Industry-standard terminology used
- [ ] **Company-specific terms adapted** - Avoided forced matches for unfamiliar processes
- [ ] **Keyword density appropriate** - Not over-optimised or unnatural
- [ ] **Interview authenticity maintained** - Every claim can be confidently discussed

### **Role-Specific Alignment**
- [ ] **Core competencies addressed** - All major JD requirements covered
- [ ] **Team management scope** - Appropriate team sizes and leadership examples
- [ ] **Industry experience** - Relevant sector and client examples included
- [ ] **Geographic relevance** - Local market context where applicable

## Technical Formatting Checks

### **Document Structure**
- [ ] **Professional summary** - Concise, relevant, keyword-rich
- [ ] **Key achievements** - 6-8 greatest individual hits with strong metrics
- [ ] **Professional experience** - Chronological, complete, properly formatted
- [ ] **Education & certifications** - Current, relevant, properly formatted
- [ ] **Technical competencies** - Aligned with role requirements

### **Consistency Verification**
- [ ] **Date formats consistent** - Month/Year format throughout
- [ ] **Company names consistent** - Exact names used consistently
- [ ] **Job titles consistent** - Matches database entries
- [ ] **Formatting consistent** - Bullet points, bold text, spacing

### **Readability Checks**
- [ ] **Concise bullet points** - Clear, impactful, results-focused (target 2 lines maximum)
- [ ] **Length discipline** - 3 lines only where additional detail enhances ATS keywords or human reader impact
- [ ] **No redundancy** - Each bullet adds unique value
- [ ] **Logical flow** - Information presented in optimal order
- [ ] **Professional tone** - Appropriate for senior role level

## Pre-Conversion Verification

### **Content Completeness**
- [ ] **All required sections present** - No missing standard CV elements
- [ ] **Contact information current** - Accurate details provided
- [ ] **LinkedIn profile aligned** - Consistent with CV content
- [ ] **References available** - Can support all major claims

### **Final Review Process**
1. **Read complete CV aloud** - Check flow and natural language
2. **Verify every quantified claim** - Trace back to database source
3. **Test ATS keywords** - Ensure natural integration
4. **Check interview readiness** - Prepare STAR stories for major bullets
5. **Validate against JD** - Confirm all requirements addressed

## Conversion Quality Checks

### **Post-Conversion Verification** (if converting to Word)
- [ ] **Formatting preserved** - No broken layouts or spacing issues
- [ ] **Font consistency** - 10pt Calibri applied throughout
- [ ] **Bullet points aligned** - Proper indentation maintained
- [ ] **Page breaks appropriate** - No awkward splits or orphaned content
- [ ] **Headers/footers correct** - Professional appearance maintained

### **Final Sign-Off Criteria**
- [ ] **Database integrity maintained** - All content traceable and authentic
- [ ] **ATS optimisation complete** - Keywords naturally integrated
- [ ] **Interview confidence high** - Can discuss every bullet point confidently
- [ ] **Role alignment strong** - Addresses all major JD requirements
- [ ] **Professional presentation** - Ready for submission

## Common Issues to Avoid

### **Critical Errors**
- ❌ **Merging different experiences** - Each bullet must be single exp_id
- ❌ **Inventing metrics** - Only use exact database figures
- ❌ **American English spelling** - Must use Australian variants
- ❌ **Title case headings** - Must use sentence case formatting

### **ATS Pitfalls**
- ❌ **Keyword stuffing** - Unnatural overuse of JD terms
- ❌ **Forced company terms** - Using unfamiliar processes for keywords
- ❌ **Inflated descriptions** - Claims beyond actual experience level
- ❌ **Generic positioning** - Not tailored to specific role requirements
