## Key Commands

### `enterprise_architecture_strategy`
**Purpose**: Develop comprehensive enterprise architecture strategy aligned with business objectives
**Inputs**: Business strategy, current technology landscape, growth projections, competitive analysis
**Outputs**: Architecture strategy document, technology roadmap, investment priorities, governance framework
**Use Cases**: Digital transformation planning, technology strategy development, architecture modernization

### `multi_cloud_architecture_design`
**Purpose**: Design vendor-neutral, multi-cloud architecture solutions for enterprise scale
**Inputs**: Business requirements, compliance constraints, performance needs, cost parameters
**Outputs**: Multi-cloud architecture blueprint, integration patterns, governance model, migration strategy
**Use Cases**: Cloud strategy development, vendor independence, risk mitigation, optimal workload placement

### `technology_evaluation_framework`
**Purpose**: Systematic evaluation of cloud technologies and vendor solutions
**Inputs**: Evaluation criteria, vendor proposals, technical requirements, business constraints
**Outputs**: Technology comparison matrix, recommendation report, implementation roadmap, risk assessment
**Use Cases**: Technology selection, vendor evaluation, make vs buy decisions, strategic partnerships

### `architecture_governance_design`
**Purpose**: Establish enterprise architecture governance, standards, and decision-making frameworks
**Inputs**: Organization structure, technology portfolio, compliance requirements, business objectives
**Outputs**: Governance framework, architecture standards, decision matrices, review processes
**Use Cases**: Architecture standardization, quality assurance, technology governance, compliance management

### `digital_transformation_roadmap`
**Purpose**: Create comprehensive digital transformation strategy and execution roadmap
**Inputs**: Current state assessment, business vision, market analysis, organizational capabilities
**Outputs**: Transformation strategy, phased roadmap, capability development plan, success metrics
**Use Cases**: Digital transformation, modernization planning, competitive positioning, innovation strategy

### `enterprise_integration_architecture`
**Purpose**: Design complex enterprise integration patterns and API strategies
**Inputs**: System inventory, integration requirements, data flows, performance requirements
**Outputs**: Integration architecture, API strategy, data management plan, security framework
**Use Cases**: System modernization, data integration, API management, microservices transformation

### `architectural_risk_assessment`
**Purpose**: Comprehensive risk assessment of proposed architectural changes
**Inputs**: Proposed architecture, current systems, dependencies, business impact analysis
**Outputs**: Risk register, mitigation strategies, contingency plans, decision recommendations
**Use Cases**: Architecture reviews, change impact analysis, risk management, decision support
